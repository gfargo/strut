# strut full-codebase audit — 2026-07-12

Follow-up to the 2026-07-03 audit (`AUDIT-2026-07.md`, issues #206–#263). Covers
v0.37.0 (VERSION `0.37.0`, HEAD `46b9cb5`). Eight parallel passes over the whole
`lib/` tree, the `strut` entrypoint, `templates/recipes/`, `completions/`,
`install.sh`, `action.yml`, and `tests/`. Every finding below was verified by
tracing the current code (line numbers are against v0.37.0); several were
reproduced with live shell tests.

The BATS suite is green on a fully-provisioned host — the 16 local failures
observed during the audit were all environmental (no `ssh-keygen`/`openssh` in
the sandbox, `age` present where a test expects it absent, and Docker daemon
down so `doctor` legitimately exits 1).

Recurring root cause: **several subsystems still `set -a; source` env/config
files directly** even though `safe_load_env` (`utils.sh:668`) was built to stop
exactly that. This is both an RCE vector and, on legal `KEY=value with spaces`
files, a silent `exit 127`.

## P0 — crash / data-loss / security

| # | Area | Summary |
|---|------|---------|
| 1 | cli | Dispatcher `set -a; source "$ENV_FILE"` (`strut:842`) runs every env file as shell on every stack command — defeats `safe_load_env`, RCE, and `exit 127` on a legal `KEY=a b`. |
| 2 | mcp | Model-controlled tool args (`service`/`stack`/`host`) reach a remote shell unquoted via `run_remote_strut` (`utils.sh:987`) → remote command injection through auto-approved read-only tools. |
| 3 | keys | `ssh:revoke`/`ssh:rotate` match `authorized_keys` on `$NF == username`, but keys are generated with comment `username@stack` → revoke removes nothing, reports success; rotate accumulates keys. |
| 4 | backup | Neo4j `.dump` restore (`backup.sh:221`) runs `neo4j-admin database load --overwrite-destination=true` after only `[ -f ]` — a truncated/empty dump wipes the live DB then fails. |

## P2 / P3 / FEAT — roundups

- **Output/JSON robustness:** duplicate `json_escape` implementations both miss control chars; `out_table_row` collapses empty cells; `fleet status --json` can emit `"dirty":,`; `notify` leaves `event`/`key` unescaped; `out_json_close_*` underflows on an empty stack.
- **Flag-parsing & dispatch/usage/completions drift:** `audit … --sudo` captured as a positional; `migrate <host> --yes` → `vps_user="--yes"`; `group logs --env` dropped; `cert --host` ignored; `scaffold --help` creates a stack named `--help`; `scaffold` outside a project writes into `~/.strut/`; completions offer five nonexistent `keys` subcommands and omit `drift images`/`gateway`; `usage()` omits `validate`/`history`/`logs:download`/`drift images`.
- **SSH/scp option consolidation:** `provision` passes ssh `-t`/`-p` to `scp` (breaks on non-default port); `gateway`/`domain` hand-roll scp opts with `StrictHostKeyChecking=no`. One `build_scp_opts` helper.
- **Test-coverage gaps:** no tests for `cmd_webhook.sh`, `cmd_mcp.sh`/`mcp/*`, `cmd_drift_images.sh`, `keys/pull.sh`/`discovery.sh`/`status.sh`, `migrate/phase-*`; add a recipe-contract lint that would have caught #28/#29.
- **Smaller correctness:** `backup compare` counts `pg_class` rows not table rows; `drift`/`schedule` cron install over-matches sibling stacks by substring; `remote:init` ignores `DEFAULT_BRANCH` (dead `branch="main"` fallback); `logs -f` interrupt reports a false SSH error; `install.sh` `reset --hard origin/<tag>` breaks tag pins on re-run; several `[ -z "$RED" ]` guards abort under `set -u`.
