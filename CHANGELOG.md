* **ssh:** use %C hash in ControlPath to prevent sun_path overflow on macOS ([#159](https://github.com/gfargo/strut/issues/159)) On macOS, $TMPDIR is ~49 chars long (/var/folders/.../T/) and combined with the verbose %r@%h:%p pattern (especially with Tailscale *.ts.net hostnames), the SSH control socket path exceeded the BSD sun_path limit of 104 chars. This caused SSH to silently fail, surfacing as 'unreachable' errors. Changes: - Switch ControlPath token from %r@%h:%p to %C (fixed-length SHA1 hash of local host, remote host, port, user) — always 40 hex chars - Default control dir to /tmp instead of $TMPDIR (short, writable) - Update ssh_mux_cleanup to match new strut-mux-$$-* naming pattern - Add test asserting total path length stays under sun_path limit The worst-case path is now /tmp/strut-mux-99999-&lt;40chars&gt; = 62 chars, well under the 104-char limit even with OpenSSH suffixes. STRUT_SSH_CONTROL_DIR override and STRUT_SSH_NO_MUX=1 opt-out still work.
